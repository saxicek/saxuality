--- layout: post title: Oracle Designer - working directly with repository tables date: '2008-09-05T11:40:00.002+02:00' author: Libor Nenad√°l tags: - Oracle Designer modified_time: '2008-09-05T11:53:07.311+02:00' blogger_id: tag:blogger.com,1999:blog-17453343.post-5203282874327838822 blogger_orig_url: http://saxuality.blogspot.com/2008/09/oracle-designer-working-directly-with.html --- Today I needed to compare 2 application systems from Oracle Designer and as I got over 100 entities and don't like the embedded Java comparative tool I decided to compare it directly in the repository database. Good schema introduction is here. And here is my solution:
  1. You need to set workarea context:
    begin
      jr_context.set_workarea('GLOBAL SHARED WORKAREA');
    end;
    /
  2. And then just compare entities from 2 applications:
    WITH entity_system AS (
    SELECT e.NAME entity
          ,s.NAME system
      FROM repadm.ci_entities e
      JOIN repadm.ci_app_sys_entities se ON se.entity_reference = e.id
      JOIN repadm.ci_application_systems s ON s.id = se.application_system_reference
    )
    SELECT t.entity
          ,t.system
          ,decode(t2.entity, t.entity, 'OK', 'missing')
      FROM entity_system t
      LEFT OUTER JOIN entity_system t2 ON t.entity = t2.entity
                                      AND t2.system = decode(t.system
                                                            ,'Application_1'
                                                            ,'Application_2'
                                                            ,'Application_1')
     WHERE t.system IN ('Application_1', 'Application_2');
