I performed little testing of Google Fusion Tables today and while it is quite nice technology which allows very easy import of large data sets, it is still very immature. I tested with data from http://www.psp.cz/sqw/hp.sqw?k=1300 and found following issues:
  1. File names are restricted. When I tried to load *.unl file, it was refused with error than only CSV, TSV, ... extensions are allowed. I had to rename the files to CSV.
  2. It is hard so set file encoding. The list of available encodings is too long and flows out of the page. As a workaround I converted files to UTF8 (were Windows-1250 originally).
  3. When renaming columns I had to do it in 2 phases as I was not able to rename last column. Workaround is to rename all columns except the last one, save and then rename the last one.
  4. Cannot merge several tables using different keys. That is quite an issue if you have non-trivial schema. The issue is logged in https://code.google.com/p/fusion-tables/issues/detail?id=766 and hopefully it gets fixed soon as developer is assigned to it.
Summary: It is a nice technology but needs to be enhanced to be used to something more than just simple load and 2 tables merge. API access (which is supported but I have not tested it) could be used to nice mash ups.
